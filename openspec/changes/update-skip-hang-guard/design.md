## Context
Переходы блоков могут зависать при симуляции PLC и коротких импульсах End/Error, если ожидание строится только на подписках.

## Goals / Non-Goals
- Goals: фиксировать Skip при коротких импульсах; исключить вечное ожидание перехода блока; информировать оператора.
- Non-Goals: менять итог теста на NOK/Failed; изменять логику PLC; добавлять новые режимы Retry.

## Decisions
- Решение: перед ожиданиями по подписке выполнять прямое чтение BlockEndTag и BlockErrorTag.
- Решение: при таймауте ожидания "все колонки idle" (10с) ставить паузу и уведомлять оператора без NOK.
- Решение: при ожидании сброса Skip сигналов выполнять прямое чтение и выходить без ожидания, если теги уже false.

## Risks / Trade-offs
- Возможны расхождения между подпиской и прямым чтением; снижаем риск, требуя одновременного true по двум тегам.
- Таймаут 10с может быть недостаточным для отдельных стендов; в рамках изменения оставляем фиксированным.
