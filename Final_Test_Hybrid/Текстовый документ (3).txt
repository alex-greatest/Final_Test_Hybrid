6)	Старт операции. POST http://185.222.161.44:8080/api/operation/start. Post запрос.

Старт операции

Тело запроса:

{
  "serialNumber": "12F15770000498732304313",
  "stationName": "final_test",
  "operator": "5555",
  "admin": ""
}

admin в данном случае не обязателен, можно просто пустую строку
Если котёл на правильном этапе процесса, то придёт вот такой ответ:
 

boilerMadeInformation – информацию о заказе и котле


orderNumber – номер заказа
amountBoilerMadeOrder – сколько котлов в заказе сделано
amountBoilerOrder – кол-во котлов в заказе
correlationId – это игнорируй, нужно было для других приложений

boilerTypeCycle – информация о котле 

typeName – тип
article – артикул
serialNumber – серийный номер
model – модель


Это всё просто для инфо, можно вывести на экран


"recipes": [
    {
      "parameter": "Plant_ID",
      "value": "0",
      "plcType": "STRING"
    },

Рецепты – это список. Каждый элемент, это объект с параметром.

Если рецептов нет для этого типа, то ответит соответствующе. 404 статус  и объект с message. 


Если котёл не на том этапе процесса, то придёт такой ответ
 

http статус 403, и json с ошибкой, где перечислена инфа и для какого котла.

Администратор (бригадир или мастер) можно разрешить прохождение станции. Надо сделать запрос
POST http://185.222.161.44:8080/api/operation/rework 

Тело запроса тоже самое, но надо ещё добавить админа. Для этого надо сделать запрос на сервер, чтобы получить инфу об админе.
 Оператора тоже надо обязательно

{
  "serialNumber": "EngP3080920250000048732302142",
  "stationName": "final_test1",
  "operator": "5555",
  "admin": "master"
}

Ответит статусом 200 
 

И после этого нужно повторить запрос на старт операции.
